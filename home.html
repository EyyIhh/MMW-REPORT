Introduction

Linear programming is an optimization technique for a system of linear constraints together with a linear objective function. The objective defines the quantity to be optimized (maximize or minimize) and the goal is to find variable values that optimize that objective.

Example (informal): A factory produces two items (doodads and whirligigs) with cost and time constraints. Using linear programming we model constraints as linear inequalities and an objective (profit), then find the production quantities maximizing profit.

When to use Linear Programming

Use linear programming when:

  • The goal is to maximize or minimize some quantity.
  • All constraints can be written as linear inequalities.
  • The objective function is linear in the decision variables.

A constraint is an inequality that defines how the values of the variables in a problem are limited. In order for linear programming techniques to work, all constraints should be linear inequalities.

Factory example (constraints): 
	2x + 4y ≤ 220  
	3x + 2y ≤ 150  
	x ≥ 0  
	y ≥ 0

Objective (profit): P = 4x + 3y (maximize).

Linear Programming in Two Variables

In two-variable LP problems the feasible region is a convex polygon and an optimal value of a linear objective occurs at a vertex of that polygon.

Theorem (vertex optimality): Given a convex polygonal feasible region and a linear objective \(f(x,y)=ax+by\), any maximum or minimum occurs at one of the region’s vertices.

Practical procedure:

  1. Confirm feasible region is a convex polygon and objective is linear.
  2. Find coordinates of each vertex of the feasible region.
  3. Substitute each vertex into the objective function; choose the vertex giving the highest (or lowest) value.
Sample feed-mix problem (summary): Decision variables: kilograms of corn (c) and wheat (w). Constraints and objective can be written as linear inequalities and a linear cost function. Solving vertices and evaluating the cost yields the minimizing mix.

Simplex Algorithm (Overview)

For linear programming problems with many variables and constraints, the simplex algorithm is a systematic method to move from one vertex (basic feasible solution) of the feasible polytope to an adjacent vertex with a strictly better objective value, and repeat until an optimum is reached.

Key ideas:

  • Convert the problem to standard form (equalities with slack variables, nonnegative variables).
  • Identify a basic feasible solution (often by setting nonbasic variables to zero and solving for basics).
  • Pivot: choose an entering variable to improve objective, and a leaving variable to maintain feasibility.
  • Repeat until no entering variable improves the objective (optimality condition); or detect unboundedness / special cases.

Special Cases of the Simplex Algorithm

While using simplex, several special cases may occur:

  • Degeneracy: A basic feasible solution has one or more basic variables equal to zero. Can cause cycling; anti-cycling rules (Bland’s rule) mitigate this.
  • Unboundedness: The objective can be increased without bound while remaining feasible — no optimal finite solution.
  • Infeasibility: No feasible solution exists satisfying all constraints.
  • Multiple optimal solutions: If an objective line is parallel to a constraint side, all points on that side (including vertices) may achieve the same optimal objective value.

Big-M Method (Summary)

When the initial system does not present an obvious basic feasible solution (for example, equations include ≥ constraints or equalities without slack variables), the Big-M method introduces artificial variables with a large penalty (M) in the objective.

The artificial variables are driven to zero by the penalty M if a real feasible solution exists. After constructing the augmented system and objective, simplex is applied. If artificial variables remain positive in the final solution, the original problem is infeasible.

Linear Inequalities

Linear inequalities are expressions in which two linear expressions are compared using inequality symbols such as <, >, ≤, ≥, or ≠.

The basic symbols used in linear inequalities are:

  • Not equal (≠) – e.g., x ≠ 3
  • Less than (<) – e.g., x + 7 < √2
  • Greater than (>) – e.g., 1 + 10x > 2 + 16x
  • Less than or equal to (≤) – e.g., y ≤ 4
  • Greater than or equal to (≥) – e.g., −3 − √3·x ≥ 10

Solving Linear Inequalities (One Variable)

The process for solving linear inequalities is the same as for linear equations, but one must pay attention to the inequality symbol (especially when multiplying/dividing by a negative number).

For example: 

2x < 6 ⇒ x < 3
Or if multiplying/dividing by negative: 
−2x + 3 > 6 ⇒ −2x > 3 ⇒ x < −3/2
Notice the inequality flips when dividing by −2.

Understanding Open and Closed Intervals

When representing the solution of a linear inequality, we often express it as an interval. An interval shows all real numbers between two endpoints, possibly including or excluding the endpoints.

  • Open interval (a, b): All numbers between a and b, but not including a or b.
    Example: (2, 5) means 2 < x < 5.
  • Closed interval [a, b]: Includes both endpoints.
    Example: [2, 5] means 2 ≤ x ≤ 5.
  • Half-open (or half-closed) interval: Includes one endpoint but not the other.
    Example: [2, 5) means 2 ≤ x < 5.
Note: Parentheses ( ) denote open endpoints, while brackets [ ] denote closed endpoints.

Representing Inequalities on a Number Line

Solutions of inequalities are often visualized on a number line for clarity. A shaded line or arrow shows all numbers satisfying the inequality, while open or filled circles mark whether endpoints are included.

  • Open circle: Endpoint not included (< or >).
  • Closed circle: Endpoint included (≤ or ≥).
  • Arrow direction: Indicates where the inequality extends (left for smaller numbers, right for larger).
Example: For x ≥ 3, place a closed circle at 3 and shade the line to the right. For x < −2, place an open circle at −2 and shade to the left.

These visualizations make it easier to understand ranges of values that satisfy inequalities and prepare the foundation for understanding solution regions in linear programming problems.

Graphing Linear Inequalities (Two Variables)

For a linear inequality in two variables (e.g., Ax + By > C), the solution set is a region (half‐plane) rather than a single line.

Key steps:

  • Rewrite as y = … (if needed) to identify the boundary line.
  • Plot the line: Use a **solid line** if the inequality is ≤ or ≥; use a **dashed line** if it is < or >.
  • Test a point not on the line (often the origin (0,0)) to see which side of the line satisfies the inequality; then shade that half‐plane.

Examples of Linear Inequalities in Two Variables

Here are some typical forms: 

		  2x < 3y + 2  
		  7x − 2y > 8  
		  3x + 4y + 3 ≤ 2y − 5  
		  x + y ≥ 0

Difference, Importance & Real-Life Examples

Difference – Linear Programming vs Linear Inequalities

While both linear programming and linear inequalities deal with linear expressions and constraints, they differ in key ways:

  • Focus: Linear inequalities focus on describing a region of solutions satisfying one or more inequality constraints (e.g., “\(x + 2y ≤ 10\)”). Linear programming uses inequalities (and/or equalities) plus an objective function (maximize or minimize) to find the *best* solution among those feasible ones.
  • Solution set: With linear inequalities, the solution is typically a region (in 2 variables a half-plane or polygon) of all points satisfying the inequalities. : In linear programming, you look for a particular optimal point (often a vertex of that feasible region) that maximizes or minimizes the objective.
  • Objective: Linear inequalities often do not include an objective to optimize — they simply specify constraints. Linear programming always includes an objective function (e.g., profit, cost).
  • Graphs: Graphing a linear inequality involves shading a region. Graphing a linear programming problem uses that same region but then overlays “level” lines of objective function and finds where it touches the region’s boundary.

Importance

Understanding both topics is important because:

  • Linear inequalities provide the language and structure of constraints — without them you cannot define feasible regions or restrictions in real world models.
  • Linear programming builds on those constraints to help decision-makers optimize resources (time, money, materials) under those constraints.
  • Many real life problems are best framed first as inequalities (“you cannot exceed budget”, “you must meet minimum production”) and then as programming problems (“maximize profit under budget and production constraints”).

Real‐Life Examples

Here are some concrete applications:

  • Diet planning: Suppose you must ensure daily intake of certain nutrients (inequalities) and minimize cost while meeting those constraints → linear programming.
  • Resource allocation for factories: You may have labor hours and machine hours constraints (inequalities) and you want to maximize output or minimize cost → linear programming.
  • Budget limits: You have income and expenses must not exceed income (inequality) so you find acceptable expenditure combinations.
  • Shipping and logistics: Limiting weights, volumes, capacities (inequalities) and choosing shipping mix to minimize cost (linear programming).
